# 📅 Day 03 - Understanding the Operating System

## 🎯 Goal

Understand what an operating system really is before learning Linux commands.

---

## 📚 What I Learned

- An operating system is much more than a graphical interface.
- The operating system acts as a resource manager between applications and hardware.
- The kernel is the core of the operating system.
- Applications do not communicate directly with the hardware.
- The kernel manages CPU, memory, storage, and devices.
- The kernel also protects applications from interfering with each other.
- CPU time is shared between multiple programs using scheduling.
- Multitasking is achieved by switching between tasks very quickly.

---

## 💻 What I Did

- Explored the role of an operating system.
- Learned the difference between the operating system and the kernel.
- Understood why applications should not access hardware directly.
- Learned the basic idea of CPU scheduling.

---

## ❌ Mistakes I Made

- I thought the CPU would simply crash if programs accessed it directly.
- I didn't know the kernel is the actual component that manages hardware resources.

---

## 💡 Biggest Lesson

The kernel is the bridge between software and hardware.

Without it, applications could not safely share computer resources.

---

## 🧠 Reflection

Today I learned that every application depends on the kernel to access hardware.

The operating system is not just a desktop environment; it is the manager that keeps the whole computer organized and secure.
