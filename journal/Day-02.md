# 📅 Day 02 - Why Linux?

## 🎯 Goal

Understand why Linux is the dominant operating system for servers before learning any Linux commands.

---

## 📚 What I Learned

- DevOps is not about using tools; it's about solving real problems.
- Linux became the standard for servers because of its architecture, efficiency, and flexibility.
- Most servers do not need a graphical interface (GUI).
- A server is designed to provide services, not to be used like a personal computer.
- Every technical decision also has an economic impact.
- Reducing resource usage on thousands of servers can save a company millions of dollars.
- Servers are usually managed remotely over a network instead of using a monitor, keyboard, and mouse.
- SSH is the standard way to communicate with Linux servers remotely.

---

## 💻 What I Did

- Discussed why Linux is preferred over Windows for servers.
- Compared GUI-based operating systems with command-line server environments.
- Learned that removing unnecessary components improves efficiency.
- Understood that servers can be managed remotely without a monitor.
- Connected the concepts of networking, IP addresses, and remote server management.

---

## ❌ Mistakes I Made

- I thought Linux was originally created specifically for servers.
- I assumed Linux with a GUI would become similar to Windows.

---

## 💡 Biggest Lesson

A server does not need to be visually attractive.

Its job is to be stable, efficient, secure, and available all the time.

---

## 🧠 Reflection

Today I realized that a server doesn't need a screen to be managed.

What matters is the ability to communicate with it over the network.
