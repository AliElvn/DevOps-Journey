# 📅 Day 01 — Beginning the Journey

## 🎯 Goal

Start my DevOps journey and set up my learning environment.

---

## 📚 What I Learned

- What DevOps really means.
- DevOps is not just the bridge between developers and networking.
- Difference between Git and GitHub.
- Difference between `git add`, `git commit`, and `git push`.
- Why Git needs `user.name` and `user.email`.
- The concept of Local Repository and Remote Repository.

---

## 💻 What I Did

- Created the DevOps-Journey repository.
- Added the roadmap.
- Updated the README.
- Cloned the repository to my computer.
- Configured Git for the first time.
- Created the `journal` folder.
- Made my first real Git commit.

---

## ❌ Mistakes I Made

- I thought DevOps was only a bridge between developers and networking.
- I tried to commit before configuring Git identity.
- I didn't understand why `git add` and `git commit` are separate steps.

---

## 💡 Biggest Lesson

Git doesn't automatically save everything.

It first lets me choose what should be included, then creates a snapshot with a commit, and finally uploads it to GitHub using push.
